# phyl-sched

Scheduler service. Scans `$PHYLACTERY_HOME/schedule/` every 5 seconds and fires due entries by creating sessions via the daemon API.

## How It Works

1. Every 5 seconds, read all `.json` files in `schedule/`
2. Parse each as a `ScheduleEntry`
3. If `entry.at <= now`, create a session via `POST /sessions` on the daemon
4. On success, delete the schedule file
5. On failure, leave it for retry on the next scan

## Error Handling

- **Corrupt files** (invalid JSON): renamed from `.json` to `.bad` so they don't block the scan loop
- **Failed API calls** (daemon down, etc.): entry is left in place and retried every 5 seconds
- **`.tmp` files**: ignored (these are in-flight atomic writes from `phyl-tool-self`)

## Running

Started unconditionally by `phyl start --all`. Also available as:
- systemd: `phyl-sched.service` (generated by `phyl setup systemd`)
- launchd: `com.phylactery.sched.plist` (generated by `phyl setup launchd`)

Can also be run standalone:

```bash
phyl-sched
```

Requires `$PHYLACTERY_HOME` or the default home path to exist with a valid `config.toml`.

## Configuration

No specific configuration. Uses `[daemon].socket` from `config.toml` to connect to the daemon.

## Shutdown

Handles `SIGINT` (Ctrl-C) gracefully. Completes the current scan cycle, then exits.
